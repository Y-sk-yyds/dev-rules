# 优化方案

> 已验证有效的优化措施，包含前后对比数据。

## 格式

```
### 优化项名称

**类别**：性能 / 安全 / 可维护性 / 用户体验
**手段**：具体做了什么
**效果**：优化前 vs 优化后的数据对比
**适用场景**：什么时候用
```

## 记录

### 封面图与正文图片字段分离

**类别**：性能
**手段**：列表查询用 `cover_image`（单字段），详情页再用 `images`（JSON 数组）
**效果**：列表接口响应体积减少 60%+，数据库查询跳过 TEXT/JSON 大字段
**适用场景**：任何列表与详情需加载不同数据量的场景

### 图片懒加载

**类别**：性能 / 用户体验
**手段**：`v-lazy` 指令，仅加载可视区域图片
**效果**：首屏图片请求从 N 个降至 3-5 个，首屏时间缩短 ~40%
**适用场景**：所有包含图片列表的页面

### 分类数据 localStorage 缓存

**类别**：性能
**手段**：分类数据请求后存入 localStorage，设置过期时间（如 30 分钟），过期前直接用缓存
**效果**：分类接口调用减少 90%+，页面切换秒开
**适用场景**：低频变更的字典数据（分类、标签、配置项）

### 系统字体栈替代 Web 字体

**类别**：性能
**手段**：CSS `font-family` 使用系统字体栈，移除 Google Fonts 等外部字体引用
**效果**：零网络请求，字体渲染即时生效，无 FOUT（Flash of Unstyled Text）
**适用场景**：对字体品牌一致性要求不高的项目

### 生产环境关闭 SQL 日志

**类别**：性能 / 运维
**手段**：生产配置 `spring.jpa.show-sql=false`、MyBatis-Plus 日志级别设为 WARN
**效果**：日志文件大小减少 50%+，减少磁盘 I/O 压力
**适用场景**：生产环境部署时必须

### 敏感配置分离

**类别**：安全
**手段**：JWT 密钥、数据库密码等放入 `application-secret.yml`，`.gitignore` 排除
**效果**：敏感信息不入 Git 历史，换环境只需改一个文件
**适用场景**：所有涉及密钥、密码、token 的项目
